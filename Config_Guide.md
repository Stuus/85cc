# 85度C 日結匯報系統 - 設定檔 (config.json) 說明指南

本系統的核心在於高度的「自定義設定檔」。您可以在系統的「設定頁面」透過視覺化介面操作，系統內建了強大的**「查詢表達式」**，幫助您精準從銷售日報表中抓出任何您需要的數字。

---

## ⭐️ 核心語法：查詢表達式 (Query Expression)

您在設定頁面中填寫的「查詢表達式」，必須遵循以下格式：
`前綴符號` + `匹配符號與關鍵字` + `結尾符號`

### 1. 前綴符號 (尋找目標)
- `>`：尋找**類別** (對應銷售日報表最左側的「類別」欄位，通常會抓取該類別的「合計」列)。
- `~`：尋找**品項** (對應銷售日報表中的「品項」欄位)。

### 2. 匹配符號與關鍵字
- `{關鍵字}`：**完全一樣** (精確比對)。例如 `{長條1}`。
- `[關鍵字]`：**含有即可** (模糊比對)。例如 `[蛋糕]`，只要名稱包含蛋糕就會被抓出來加總。

### 3. 結尾符號 (抓取數值)
- `:A`：抓取該列的**「銷售量」** (數量)。
- `:B`：抓取該列的**「銷售金額」** (營收)。

### 4. 特殊符號 (手動輸入)
- `*`：如果您將查詢表達式設定為單一個 `*`，系統就不會去 Excel 裡面抓資料，而是會在「製作日報」的首頁**自動產生一個手動輸入框**，讓您自行手寫數值（例如：試喝品項）。

### 💡 進階用法：算式運算
您可以把多個表達式串在一起進行加減運算！
例如：`>[飲料]:B - >[咖啡]:B` (飲料類別的總金額 減去 咖啡類別的總金額)。

## 各設定區塊說明

### 1. 各分類營業額 (categories)
用來設定在簡訊上方顯示的各分類營收。由於是營收，通常結尾符號會使用 `:B` (銷售金額)。
- **範例 1**：`>[長條1]:B + >[整膜1]:B` (把長條1跟整膜1兩個類別的銷售金額加總，當作蛋糕營收)
- **範例 2**：`>[飲料]:B - >[咖啡]:B` (飲料營收扣除咖啡營收)

### 2. 追蹤單一品項銷售 (trackSales)
用來追蹤特定商品的銷售數量。結尾符號通常使用 `:A` (銷售量)。
- **範例 1**：`~[芙蓉蛋塔]:A` (只要品項名稱含有芙蓉蛋塔，就把銷售量加總)
- **範例 2**：`~{993-慢熟千層蛋糕(條)}:A` (精確比對這個品名，抓出銷售量)
- **範例 3**：`~[長條]:A` (名稱含有長條的所有商品加總)

### 3. 組合顯示品項 (combinedSales)
用來將兩個以上的商品合併顯示在同一行，中間用斜線 `/` 分隔。
- 在設定中，您可以點擊新增多個「查詢表達式」，例如第一個填 `~[杜拜法國]:A`，第二個填 `~[杜拜Q餅]:A`。
- 輸出結果會長這樣：`杜拜法國/Q餅：20/102`。

### 4. 庫存盤點項目 (inventoryItems)
定義每天需要回報的盤點清單。
- 勾選「需標註日期」：首頁會變成一個「動態清單」，允許您添加多筆包含「日期」與「數量」的紀錄（例如因應保存期限，不同日期的鮮奶各有幾瓶）。輸出簡訊時，會自動幫您按照日期（先到後）進行排列。

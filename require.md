# 工具說明

`85度C` 台北景美店日結匯報資訊輔助整合工具。此工具旨在加速晚班員工的日結流程，自動讀取報表並結合手動輸入的數據，快速生成可用於簡訊或 Line 回報的純文字格式。

## 系統輸入

- **檔案自動輸入**：
  - `現金日報表.xls`（舊版 Excel 格式）
  - `銷售日報表.xlsx`（新版 Excel 格式）
- **手動輸入項目**：
  - 盤點內容（如：庫存的鮮奶日期與數量、豆類數量、奶精、多多等）。
  - 當日班表時數（工時）。
  - 值班人員姓名。
  - 其他無法由系統自動獲取的特定銷售或報廢數據。

## 系統輸出

- **輸出格式**：`[日期]回報簡訊.txt`（如 `0509_回報簡訊.txt`）
- **輸出內容架構**：
  1. **營業額總覽**：日期、店名、總營業額及各分類（蛋糕、麵包、飲料、咖啡、點心、伴手、常溫等）營業額。
  2. **營運數據**：來客數（格式如 A/B）、工時、值班人員。
  3. **銷售重點**：特定品項的銷售數量（如：長條蛋糕、杜拜系列、特定飲品等）。
  4. **報廢統計**：麵包報廢金額、常溫報廢金額等。
  5. **庫存狀況**：鮮奶各到期日數量、咖啡豆、奶精、多多及特定季節商品（如冰粽）庫存。

## 支援平台

- **Windows 版本**：提供桌面應用程式供店內電腦操作。
- **Android / iOS 版本**：提供行動裝置版本（若需要手機操作日結）。

## 核心功能需求

1. **更新檢測**
   - 啟動時自動檢測 GitHub API 獲取最新版本進行更新。
   - 提供手動檢查與更新功能。
2. **輸出內容配置 (動態設定檔)**
   - 輸出內容（如：要追蹤哪些特定銷售品項、哪些分類）需具備可配置性。
   - 具有獨立的配置檔（如 JSON 或 YAML），可以調整簡訊輸出的欄位與追蹤品項。
   - 提供「可視化操作」的設置介面，讓使用者不需直接修改程式碼或配置檔也能調整追蹤項目。
3. **資料整合與生成**
   - 根據設定檔進行 Excel 內容萃取。
   - 將自動萃取數據與手動輸入數據合併，根據設定檔的格式進行內容輸出並儲存為 txt 檔案。
4. **歷史紀錄管理**
   - 輸出的回報內容需存在本地作為歷史紀錄。
   - 提供歷史紀錄的查看功能（UI 列表瀏覽）。
   - 提供歷史紀錄的重新生成功能：如果當天發現 Excel 抓取錯誤或輸入錯誤，可修改後重新生成並保留修改記錄。
   - 提供歷史紀錄的匯出功能。

## 畫面與使用者介面 (UI)

- **主畫面**：啟動應用程式後顯示主畫面，包含以下主要功能區塊與導航：
  - **製作日報**：載入 Excel、填寫手動資料、預覽並生成 txt。
  - **版本資訊**：顯示當前版本、檢查更新。
  - **更改配置**：提供 UI 供使用者新增/刪除要追蹤的銷售品項與庫存品項。
  - **歷史紀錄**：日曆視圖或列表視圖查看過往的回報簡訊。

## 待確認問題 (Open Questions)

以下為目前規格尚待釐清之處，請協助確認以利後續開發：

1. **Excel 檔案解析**：
   - 哪些資料可以「確定」從報表中自動抓取？（例如：營業額、各分類營收、銷售品項數量、報廢金額是否都能自動抓取？
   - Excel 報表的具體欄位與格式為何？是否需要支援同時解析舊版 `.xls` 與新版 `.xlsx` 的解析函式庫？
2. **「來客」欄位意義**：
   - 回報簡訊中的來客呈現兩個數字（例如 `156/1032` 或 `155/733`），這分別代表什麼意義？（例如：晚班來客數/全天來客數？）這兩個數字都能從 Excel 獲取嗎？
3. **組合品項顯示**：
   - 簡訊中的特定品項會用斜線分隔合併顯示（例如 `杜拜法國/Q餅：20/102` 或 `荔枝系列/試喝：9/1`），這是固定合併顯示的品項嗎？在可視化設定檔中，應該如何讓使用者配置這樣的「組合顯示」？
4. **手動輸入介面**：
   - 庫存（如鮮奶的特定日期及對應數量、冰粽的日期與數量）是否全為手動輸入？是否有預設的選項或近期日期可以供員工快速點選？
5. **版本更新與權限**：
   - 透過 GitHub API 自動獲取更新，代表此專案的 Release 將是公開的 (Public Repo) 嗎？若為私有專案 (Private Repo)，則需要處理 Token 權限認證。
6. **平台技術選型**：
   - 同時需求 Windows 與 Android/iOS 版本，是否考慮使用跨平台框架（如 Flutter, React Native, 或 .NET MAUI）來開發，以達成一次編寫多平台發布？
